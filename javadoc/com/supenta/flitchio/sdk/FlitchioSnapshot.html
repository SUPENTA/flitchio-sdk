Class FlitchioSnapshot

  • java.lang.Object
    • com.supenta.flitchio.sdk.FlitchioSnapshot
  • All Implemented Interfaces:
    Parcelable

    public final class FlitchioSnapshot
extends Object
implements Parcelable
    Object describing a consistent state of Flitchio at a given moment. It behaves partly like an ButtonEvent and partly like a JoystickEvent, in the sense that it contains information about all the buttons and all the joysticks at the same time. The snapshot should be retrieved in polling mode, once per update loop iteration. If you don't use polling mode in your app, you should rather consider registering a FlitchioEventListener to receive the equivalent ButtonEvents and JoystickEvents.

    Buttons

    All the buttons are defined in InputElement.BUTTONS. For all the methods below, you can identify a button either by passing directly a InputElement.Button instance or with the integer code of the button.

    For each button, you can check its state in the current snapshot with FlitchioSnapshot.getButtonState(InputElement.Button). The possible states are listed in InputElement.Button. You can also retrieve the current pressure with FlitchioSnapshot.getButtonPressure(InputElement.Button), in the interval [0.0 ; 1.0] (a pressure value of 0.0 means that the button is not currently pressed).

    Joysticks

    All the joysticks are defined in InputElement.JOYSTICKS. For all the methods below, you can identify a joystick either by passing directly a InputElement.Joystick instance or with the integer code of the joystick.

    For each joystick, you can access its X and Y coordinates in the current snapshot with FlitchioSnapshot.getJoystickX(InputElement.Joystick) and FlitchioSnapshot.getJoystickY(InputElement.Joystick). The value retrieved is in the interval [-1.0 ; 1.0].

    Since:
    0.5.0

    ButtonEvent, JoystickEvent
